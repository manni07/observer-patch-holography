# Particle Gap Bundle Campaign

The remaining particle work is intentionally bundled. Do not promote the
compressed `P` trunk, or change live prediction roots, until the integration gate
confirms that all relevant endpoint, matching, interval, and source-object
requirements are closed.

| Bundle | Gaps | Required result |
| --- | --- | --- |
| Electroweak root | `P` trunk, Ward Thomson endpoint, RG/matching, live adoption | Source-emitted `Delta_Th(P)` package or minimal obstruction |
| Spectrum source | charged, quark, neutrino | Shared excitation/trace-lift theorem or separated obstructions |
| QCD/Thomson backend | hadron backend, hadronic endpoint | `rho_had(s;P)` production contract or missing QCD primitive |
| Integration gate | all bundles | `promote` only if every gate closes; otherwise `keep_candidate` |

Worker packets live in `campaign_packets/`. They are deliberately short so the
Oracle completion pipeline writes a result packet instead of editing papers or
live prediction code prematurely.
